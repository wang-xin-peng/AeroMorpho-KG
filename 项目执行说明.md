# 变构飞行器知识图谱项目执行说明

## 一、作业目标对照

- 概念规模：不少于 500
- 关系规模：不少于 1000
- 评估规模：人工标注 200 概念 + 400 关系
- 应用展示：支持查询与可视化（Neo4j / 命令行问答）

## 二、环境准备

```bash
conda activate E:\conda_envs\kg_env
pip install -r requirements.txt
```

## 三、执行流水线

### 1) 端到端构建（DeepKE+OneKE + Yuan-Embedding）

```bash
python src/pipeline.py ^
  --pdf-dir "变构飞行器" ^
  --parse-dir "data/parsed" ^
  --raw-triples "data/triples_raw/triples_oneke.jsonl" ^
  --fused-triples "data/triples_fused/triples_bge.jsonl" ^
  --schema-path "config/schema.json" ^
  --oneke-model "model/OneKE" ^
  --embedding-model "model/Yuan-Embedding"
```

### 2) 生成人工评估样本

```bash
python src/evaluate.py ^
  --fused-triples "data/triples_fused/triples_bge.jsonl" ^
  --out-concepts "data/eval/sample_concepts.csv" ^
  --out-relations "data/eval/sample_relations.csv"
```

### 3) 应用展示（问答检索）

```bash
python src/app_demo.py --triples "data/triples_fused/triples_bge.jsonl" --query "变构飞行器"
```

### 4) Neo4j 导入（可视化）

先在 `.env` 配置连接信息：

```env
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=your_password
```

导入命令：

```bash
python src/load_to_neo4j.py --triples "data/triples_fused/triples_bge.jsonl"
```

## 四、推荐演示 Cypher

```cypher
MATCH (n:Entity) RETURN count(n);
```

```cypher
MATCH ()-[r]->() RETURN type(r), count(r) AS c ORDER BY c DESC LIMIT 20;
```

```cypher
MATCH (a:Entity)-[r]->(b:Entity)
WHERE a.name CONTAINS "变构飞行器" OR b.name CONTAINS "变构飞行器"
RETURN a, r, b LIMIT 50;
```
